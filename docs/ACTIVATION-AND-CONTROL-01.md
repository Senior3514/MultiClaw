# MultiClaw Activation and Control 01

## Core requirement

When MultiClaw is activated, it should feel like the behind-the-scenes company begins coming alive immediately.

## Activation model

The intended product behavior is:
- user describes the product or environment
- MultiClaw generates the company structure
- the relevant claws or agents begin activating around the work
- the system becomes useful immediately, not eventually

## Autonomy model

The product should support different levels such as:
- advisory
- operator-assisted
- fuller autonomous operation

## Control model

Even when the system is highly autonomous, the user should still be able to:
- inspect what is active
- understand what each claw is doing
- steer it
- boost it
- tighten or loosen control
- keep authority over the system

## Product standard

The experience should feel:
- immediate
- simple
- powerful
- controllable
- operationally real

## North star

Instantly create and activate a real AI company behind the scenes, while keeping control easy.
