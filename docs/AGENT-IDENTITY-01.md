# MultiClaw Agent Identity 01

## Core rule

When MultiClaw creates agents or claws for a company, they should not feel like empty temporary processes.
They should feel like real operating entities inside the generated company.

## Each generated agent should carry
- a name
- a role
- a clear identity
- scoped context
- deep understanding of the company and task domain
- enough continuity to behave coherently

## Product implication

The system should not generate shallow generic workers.
It should generate company-specific operating agents with real purpose and real contextual grounding.

## Standard

The result should feel:
- intentional
- intelligent
- contextual
- coherent
- operationally real
