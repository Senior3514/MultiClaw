# MultiClaw Attach and Run 01

## Principle

MultiClaw should be able to attach itself to what already exists and start operating.
It should not require the user to rebuild the world from scratch.

## Product meaning

The user should be able to point MultiClaw at an existing product, repo, app, VPS, or environment.
MultiClaw should then create the company shape around it and begin operating.

## Agent birth rule

When the company is generated, the user should be able to communicate with the company and its agents from the moment they are born.

## Operating rule

The system should support:
- reactive work
- proactive work
- ongoing operation inside the generated company structure

## Standard

Attach, generate, communicate, operate.
That flow should feel immediate and real.
