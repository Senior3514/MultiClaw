# MultiClaw Channel Surfaces 01

## Principle

MultiClaw should not be trapped inside a single web UI.
It should be able to communicate with the user through channel surfaces that feel natural and operational.

## First-class direction

Priority channel surfaces include:
- web
- Telegram
- later Discord and Slack

## Standard

The product should let the user interact with the company through the surface that fits the moment.
Web should not be the only doorway.

## Open-source direction

This should be built in a way that remains aligned with the open-source core and portable runtime direction.
