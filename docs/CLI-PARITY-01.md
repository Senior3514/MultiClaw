# MultiClaw CLI Parity 01

## Principle

MultiClaw should not depend on the web UI for its core operating value.

## Product meaning

The user should be able to perform the important end-to-end workflow through CLI as well, including:
- install
- configure
- start
- stop
- status
- walkthrough
- key operating actions over time

## Standard

Web UI is important.
CLI parity is also important.
The product should feel real on both surfaces.
