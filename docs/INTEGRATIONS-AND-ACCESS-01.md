# MultiClaw Integrations and Access 01

## Core requirement

MultiClaw should be approachable to everything that matters around a real product.

## Channel layer

The product should be able to connect to communication surfaces such as:
- Telegram
- Discord
- Slack
- email or notification layers
- future chat and messaging surfaces

## Model layer

The product should not be trapped inside one model vendor.
It should support many providers and many model choices.

## Access layer

The product should support normal real-world access patterns such as:
- web access
- private networking
- VPN-based access where needed
- IDP-backed authentication and authorization over time

## Product implication

MultiClaw should feel simple to use even when it can connect to many channels, models, networks, and identity systems behind the scenes.

## Standard

Everything should remain:
- clear
- secure
- easy to connect
- easy to reason about
- operationally real
