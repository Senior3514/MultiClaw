# MultiClaw Business Model

## Recommendation

Use **open core**.

## Why not fully closed

Because this project wants:
- GitHub adoption
- trust
- ecosystem growth
- community templates and role packs
- fast credibility

A fully closed model would slow adoption and reduce the chance of becoming the category leader.

## Why not fully free forever

Because real users will want:
- hosted control plane
- premium templates
- managed deployment
- team sync tools
- enterprise guardrails
- better onboarding
- support

That creates room for a clean paid layer.

## Best model

### Free / open source core
- local generator
- base templates
- starter company types
- file generation
- self-hosted usage

### Paid optional layer
- hosted UI
- premium company packs
- vertical packs
- shared control plane
- managed updates
- advanced observability
- security / policy packs
- team collaboration features

## Best go-to-market line

**Start free, become operational, upgrade only when you want scale and convenience.**

## What people will pay for

Not "AI".
They will pay for:
- speed
- structure
- less chaos
- better coordination
- better defaults
- team-scale visibility
- premium packs that save them weeks

## Possible pricing later

### Open source
Free

### Pro
Low monthly price, attractive and simple

### Team
More seats, more projects, more visibility

### Enterprise
Private deployment, policy, security, support

## Rule

The open-source core must be genuinely strong.
Do not cripple it just to force upgrades.
That would kill trust.
