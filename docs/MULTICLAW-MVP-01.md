# MultiClaw MVP-01

## What must work for the first real version

The first version must generate a useful company, not just pretty text.

## Input

A short guided wizard:

- product name
- what the product does
- target users
- main business model
- current stage
- preferred tone / culture
- priority goals
- channels in use
- tech stack
- constraints / security requirements

## Output

For each generated company:

### 1. Company foundation
- company profile
- mission
- operating principles
- leadership structure

### 2. Agent company
- CEO/Operator role
- CTO / technical lead
- product team
- growth team
- support / QA / reliability roles
- optional vertical-specific roles

### 3. Files
Generated into a project directory:
- README
- brand brief
- org chart
- operating charter
- role docs
- assignment briefs
- first 7-day plan
- first 30-day roadmap

### 4. Operational layer
- isolated team chat surface
- dashboard / status view
- starter monitor definitions
- basic cadence and escalation rules

## What to avoid in MVP

- no giant autonomous promises
- no fake magic
- no bloated role explosion
- no hard fork of OpenClaw yet
- no enterprise platform before the generator is genuinely useful

## The right MVP

**A generator that makes a small but real project company that feels alive and useful on day one.**

## Phase breakdown

### Phase 1
Project generator

### Phase 2
Agent bootstrapping and workspace generation

### Phase 3
Live coordination surfaces

### Phase 4
Templates by vertical

### Phase 5
Hosted control plane and premium packs

## Success criteria

The first generated company should be good enough that a builder says:

"I would actually use this to run my project."
