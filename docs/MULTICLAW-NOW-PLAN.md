# MultiClaw Now Plan

## What it takes to make this real here and now

### 1. Scope it correctly
Do not start by trying to rewrite or replace OpenClaw.
Start by building a generator and orchestration layer on top of it.

### 2. Build one excellent flow first
The first flow should be:

**User describes product -> MultiClaw generates a real operating company -> user can immediately use it**

### 3. Deliver concrete output, not marketing output
Day-one output should include:
- generated project company docs
- agent roles
- workspace structure
- first mission plan
- internal coordination view

### 4. Use the environment we already have
We already know how to do:
- agent role design
- team webs
- private dashboards
- OpenClaw-based operating patterns
- project isolation
- Git-backed iteration

That means this can start now as a real workspace project.

## Immediate build order

### Part 1
Brand, concept, MVP, business model

### Part 2
Project generator spec

### Part 3
Starter template packs

### Part 4
First local CLI or wizard

### Part 5
Team web / control surface for generated companies

## Hard truth

What will make this succeed is not the phrase "AGI company".
It is whether the generated output is actually good enough to be useful.

## Real success condition

A builder should be able to point MultiClaw at a project and say:

"In a few minutes, I now have a coherent AI operating company around this product."
