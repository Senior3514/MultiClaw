# MultiClaw Standing Order 001

## Order

The MultiClaw internal company is expected to keep improving the product continuously.

This is not a one-time build.
This is a living product that should be tightened, refined, hardened, and elevated constantly.

## Priority rule

When choosing the next improvement, prefer the change that most improves one of these:
- usability
- clarity
- trust
- security
- speed
- depth
- product quality under real use

## Delivery rule

Do not chase decorative work.
Ship meaningful improvements that move the product closer to a world-class open-source standard.

## Communication rule

All meaningful progress, blockers, and high-leverage next moves should continue to flow upward through clean operator-facing reporting.
