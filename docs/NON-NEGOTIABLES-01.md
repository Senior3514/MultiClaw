# Non-Negotiables 01

## Product law

This product does not win by being louder.
It wins by being better.

## Non-negotiable pillars

### 1. Security first
- secret-safe by default
- least-privilege patterns
- isolated project contexts
- guarded external actions
- auditability
- safe defaults
- policy-aware routing

### 2. Real reliability
- no fake autonomy claims
- failure-aware workflows
- retries and fallbacks
- health surfaces
- explicit blockers
- graceful degradation

### 3. Fast by design
- low-friction setup
- responsive UX
- async where needed
- good default routing
- minimal wasted steps
- compact delivery loops

### 4. Beautiful enough to feel premium
- strong visual identity
- command-center feel
- clean motion
- no low-status generic AI UI

### 5. Actually useful on day one
- generated output must be operational
- not template theater
- not empty dashboards
- not dead org charts

## Engineering standard

Every layer should be designed to answer:
- Is it secure?
- Is it fast?
- Is it understandable?
- Is it resilient?
- Is it genuinely useful?

If not, it does not ship.

## Product promise

The user should feel:
- this is powerful
- this is safe
- this is fast
- this is real
- this is under control
