# MultiClaw Orchestration Input 01

## Core rule

The user should communicate with MultiClaw at the top level.
The orchestrator layer should then take the requirement, interpret it, and route the right work into the generated company.

## Product implication

The user should not need to manually coordinate every internal moving part.
MultiClaw should absorb the intent and drive the internal orchestration.

## Standard flow

1. user expresses the goal or requirement
2. MultiClaw interprets the request
3. the orchestration layer routes the work to the right claws or agents
4. the company acts behind the scenes
5. the user receives a clear result or status

## Standard

The top-level experience should feel simple.
The internal orchestration can be deep, but the user interaction should remain clean.
