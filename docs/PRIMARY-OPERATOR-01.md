# MultiClaw Primary Operator 01

## Principle

MultiClaw should have a primary operator layer that leads the user through the product and stays available as the main point of contact.

## Product meaning

The user should not feel forced to navigate a raw swarm alone.
A primary operator should:
- orient the user
- help the user understand the current state
- stay reachable in the background
- provide a clean bridge into the wider company and its leaders when needed

## Standard

The system should feel guided, not chaotic.
There can be many agents or claws underneath, but the user should always have a clear main contact point.
