# MultiClaw Setup Categories 01

## Principle

After the base install, configuration should be grouped into clear categories.
The user should understand what belongs to the core runtime, what belongs to access, and what belongs to optional channels.

## Categories

### 1. Core runtime
Includes:
- AI provider
- model
- API key
- local vs tailscale bind

### 2. Access and networking
Includes:
- local access
- Tailscale
- future identity and access controls

### 3. Channels and integrations
Includes:
- Telegram
- Discord
- Slack
- future product integrations

## Standard

The user should not face all categories at once.
The flow should reveal them in the right order.
