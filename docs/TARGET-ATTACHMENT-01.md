# MultiClaw Target Attachment 01

## Core rule

A generated MultiClaw company should be able to attach itself to a real target.

## Valid targets

Examples of targets include:
- a specific product
- a project repository
- a workspace directory
- a VPS
- a machine
- an internal system
- an operations environment

## Product implication

The user should be able to say:
- take over this project
- operate around this VPS
- maintain this repo
- protect this machine
- improve this product

And the generated company should orient itself around that target.

## Standard

The experience should feel direct and simple.
The user points at something real.
MultiClaw builds the right company around it.
