Walkthrough

One clean path from command to deployed AI company.

Install MultiClaw, point it at the target system, let it deploy the company, and keep a clear operator path at every stage.

1. Bootstrap 
curl -fsSL https://raw.githubusercontent.com/Senior3514/MultiClaw/main/scripts/bootstrap.sh | bash
2. Configure inside the product

After the base install, MultiClaw should guide provider, runtime, skills, and deeper setup from inside the product flow.

3. Verify inside the flow

The product should verify itself cleanly instead of depending on a pile of public setup commands.

4. Steer the company

Once installed, the user should be able to operate, inspect, and refine the company from the natural MultiClaw surfaces.

After install

MultiClaw should expose the rest of the journey from inside the product and CLI naturally, without turning the public walkthrough into a wall of commands.

If you need to remove it
curl -fsSL https://raw.githubusercontent.com/Senior3514/MultiClaw/main/scripts/uninstall.sh | bash
